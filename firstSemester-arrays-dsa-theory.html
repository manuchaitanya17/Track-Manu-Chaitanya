Array Basics

An array is a fixed-size indexed structure for same-type elements

The theory frames arrays as one of the most used data structures in Java. They store elements of the same data type in sequential memory-style order and use indices for access.

That index-based access is the key idea: arrays are not just containers, they are containers with position. Once indexing matters, traversal strategy matters too.

IndexingIndexing starts at zeroJava arrays use zero-based indexing, so the first element sits at index `0`.
NatureArrays are fixed-size after creationYou choose the size during creation, and that size does not change afterward.
Creation Flow

Declaration, creation, and initialization are separate mental steps

The PDF and Java file repeatedly separate array work into declaration, creation, and initialization. That breakdown matters because beginners often blur them into one action.

You first declare the variable, then create the actual array object with a chosen size, and then initialize or access values by index. The same logic extends from one-dimensional arrays to matrices.

1D arraySingle-row storage modelUse it for sequences where each value is accessed by one index.
2D arrayMatrix-style storage modelUse nested indices like `matrix[row][col]` when the structure has rows and columns.
Operations

Traversal, length, and search patterns turn arrays into problem-solving tools

The theory introduces `length` as the built-in size property you use to control loops safely. That becomes the bridge from fixed storage into algorithms such as traversal, comparison, aggregation, and searching.

In the Java source, those basics quickly expand into running sums, permutation building, doubling arrays, max-based comparisons, linear scanning, and binary search.

Linear searchGood for unsorted arraysScan one element at a time when no ordering guarantee exists.
Binary searchGood for sorted search spacesReduce the search range by comparing against the midpoint in a sorted array.

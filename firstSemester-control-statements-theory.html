Flow Model

Control statements change the default top-to-bottom execution path

The theory introduces control flow as the sequencing system behind a Java program. Without control statements, code runs once from top to bottom. With them, the same block may be skipped, repeated, or terminated early.

From there the topic splits Java flow control into three categories: selection statements, iterative statements, and jump statements.

SelectionChoose one pathUse conditions to decide which block executes.
IterationRepeat while a condition holdsLooping constructs execute blocks multiple times under controlled rules.
Selection Statements

if-based branching and switch-based dispatch solve different readability problems

The PDF groups decision-making into the `if` family and `switch`. `if`, `if-else`, `if-else-if`, and nested `if` are useful when branches depend on flexible boolean expressions.

`switch` becomes useful when one expression is matched against a fixed set of constant cases. It improves readability when a long `if-else-if` chain becomes repetitive.

  • `default` in a switch block is optional.
  • `break` is usually used to stop fall-through once a case is handled.
  • Nested `if` means a full `if` or `if-else` lives inside another condition block.
if ladderBest for condition treesUseful when each branch has a different boolean condition.
switchBest for fixed case matchingCases must be non-duplicate and of a supported type such as int, char, enum, or String.
Loops

Choose the loop type based on when the condition is checked

The theory distinguishes `for`, `while`, and `do-while` not by capability but by usage pattern. `for` works well when iteration count is known, while `while` works better when the stop condition is driven by runtime state.

`do-while` is the exit-controlled version, which means the body runs first and the condition is checked afterward. That guarantees at least one execution.

forKnown iteration countInitialization, condition, and increment or decrement are grouped in one place.
while / do-whileUnknown iteration count`while` checks first; `do-while` checks later and therefore runs once before testing.
Jump Statements

break, continue, and return transfer control in different ways

`break` exits the current loop or switch. `continue` skips the remaining statements in the current iteration and jumps to the next cycle. `return` exits the current method entirely and optionally hands back a value.

The source PDF also highlights a subtle point: in `for` loops, `continue` still moves to the increment step, but in `while` loops you must manage increment logic carefully or risk an infinite loop.

breakStop the current loop or switchOnly the nearest enclosing loop or switch is affected.
returnExit the method, not just the loopOnce `return` runs, the rest of that method body is ignored.

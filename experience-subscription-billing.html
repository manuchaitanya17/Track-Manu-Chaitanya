Runtime Surface

What the Subscription Billing Package Does

The project is compact but coherent: it creates subscription records from order lines, computes renewal dates, and produces invoices automatically when a subscription is due.

User Event

Reads subscription-marked order lines and creates custom subscription records.

Frequency Library

Looks up frequency records and computes the next billing date.

Map/Reduce

Groups due subscriptions and creates one invoice per customer + order + next date.

Grouped JSON

Development JSON artefacts show the grouping and context used while building the logic.

Custom Record

The subscription record becomes the durable state holder for renewal flow.

Business Value

Removes repeated manual renewal-order creation and standardizes billing cadence.

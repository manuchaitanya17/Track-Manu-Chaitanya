Variables

Variables are named memory slots with strict declaration rules

The theory starts by treating a variable as a named memory location that stores values during program execution. Java requires declaration before use, so the type and identifier are both part of the contract from the start.

It also emphasizes naming discipline: identifiers may contain letters, digits, `_`, and `$`, but they cannot begin with a digit, cannot contain spaces, and cannot reuse Java keywords. That basic correctness matters because the compiler enforces it early.

DeclarationThree common declaration patternsDeclare only, declare and initialize, or declare multiple variables of the same type in one statement.
RuleIdentifiers are case-sensitive`rate`, `Rate`, and `RATE` are different names, so naming consistency is part of correctness.
Data Types

Primitive values store directly, references point elsewhere

The PDF separates Java data representation into primitive types and reference types. Primitive variables hold their raw value directly, while reference variables hold the address-like reference used to locate an object in memory.

From `int`, `byte`, `short`, and `long` to `float`, `double`, `boolean`, and `char`, the theory focuses on range, storage size, and the default behavior you need to understand before writing real expressions.

  • Use `float` with an `f` suffix for literals like `12.3f`.
  • `char` is a 16-bit Unicode-based type.
  • `boolean` models program flow decisions through `true` and `false`.
PrimitiveEight built-in primitive typesThey are not objects and are chosen when you want direct value storage with predictable size and range.
ReferenceReferences point to heap objectsThe variable does not store the full object value; it stores the reference used to reach that object.
Conversion

Compatibility decides whether conversion is automatic or manual

Automatic type conversion happens when source and destination types are compatible and the destination type is wide enough to hold the original value safely. That is why an `int` can widen to `float` without an explicit cast.

Manual conversion, or casting, is required when you deliberately move into a smaller or incompatible target type. This is where truncation, precision loss, or sign changes become part of the programming decision.

WideningCompiler-driven conversionExamples include `byte` to `int`, `int` to `long`, and `float` to `double`.
CastingProgrammer-driven conversionExamples like `(int) 12.3f` show how the fractional part is explicitly discarded.
Operators

Operators define how expressions are grouped and evaluated

The last part of the theory groups operators by category: unary, arithmetic, shift, relational, bitwise, logical, ternary, and assignment. The point is not just memorization, but understanding how operator choice changes the meaning of an expression.

Precedence and associativity decide evaluation order. Higher-precedence operators bind first, while operators of the same precedence rely on left-to-right or right-to-left associativity unless parentheses force a different grouping.

PrecedenceHigher precedence binds firstMultiplication happens before addition unless parentheses intentionally override the order.
AssociativitySame precedence still needs a ruleWhen operator priority matches, associativity decides whether evaluation moves left-to-right or right-to-left.

What is in the Hugo Package

The package is small but precise. It keeps the active invoice layout beside a backup copy so changes to document structure can be compared and rolled back safely.

invoice.xml

The active invoice template carrying the current macro, table, and branding structure.

invoiceBackup.xml

A preserved backup version used for rollback and side-by-side layout comparison.

Header and Footer Macros

The XML shows reusable document framing rather than one flat markup block.

Branding Layer

Subsidiary identity, address blocks, and print-safe finance presentation are handled in markup.

Version Safety

Keeping the backup beside the active file makes format changes easier to test and reverse.

Delivery Style

This is document-surface customization work, not runtime SuiteScript automation.

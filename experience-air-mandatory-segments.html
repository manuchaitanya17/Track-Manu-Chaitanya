Conditional Segment Enforcement

AIR - Mandatory Segments Based on Account enforces Department, Location, and Market Sector based on the selected GL account. The project combines an FDD, a technical document, a client script for UI validation, a Suitelet that publishes account rules, a user event for CSV validation, and object XML for the account-level configuration fields.

Conditional Segmentation Control Across Transactions

This product is all about context-aware validation. Instead of making segments globally mandatory, it enforces them only when the selected account says they should be required, while still respecting custom-form precedence.

Control Surface

What the AIR Package Covers

The project is compact in script count but broad in impact because it affects multiple transaction types and both UI and CSV-import contexts.

Account Flags

Three account-level fields drive Department, Location, and Market Sector requirements.

UI Worker

A Suitelet publishes the account and item rule maps to the client script.

Client Validation

The client script checks supported transactions during user interaction and save.

CSV Validation

The user event reproduces the same logic during CSV imports.

Journal Skip

A journal checkbox allows explicit bypass in approved import scenarios.

Business Value

Improves posting accuracy without forcing segments globally on every transaction.
